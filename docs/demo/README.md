# ShelfOps Demo Package

This folder contains the active demo and walkthrough assets that still map directly to the current live runtime and sign-off flow.

## Files
- `AUDIENCE_PLAYBOOK.md`: How to tailor the same truth to hiring managers, technical interviewers, and SMB pilot conversations.
- `BUSINESS_WALKTHROUGH.md`: Canonical all-in-one business teleprompter. Includes actions, narration, sourced business framing, and proof steps.
- `TECHNICAL_WALKTHROUGH.md`: Canonical all-in-one technical teleprompter. Includes actions, narration, stack reasoning, commands, and MLOps proof steps.
- `DEMO_RUNBOOK.md`: Runtime setup and execution guide. Use this for environment prep, not as the main speaking script.
- `DEMO_SIGNOFF_CHECKLIST.md`: Strict final checklist for challenger evidence, visual review, polish, and rehearsal.
- `backend/scripts/prepare_demo_runtime.py`: Resets the live demo to a deterministic state.
- `backend/scripts/run_demo_terminal_showcase.py`: Read-only terminal proof for the local demo runtime.
- `CLAIMS_LEDGER.md`: Claim-to-evidence map. Keep this active so the live demo and sign-off checklist stay claims-safe.
- `SLIDE_DECK_OUTLINE.md`: 12-15 slide narrative for hiring managers and operators.
- `VIDEO_SCRIPT_10MIN.md`: Time-boxed narration script aligned to the runbook.

## Intended Use
1. Read `AUDIENCE_PLAYBOOK.md` and choose the audience framing before touching slides.
2. Choose `BUSINESS_WALKTHROUGH.md` or `TECHNICAL_WALKTHROUGH.md` as the canonical teleprompter.
3. Use `DEMO_ONE_PAGE_CHEAT_SHEET.md` only as presenter memory support after learning the main script.
4. Use `DEMO_SIGNOFF_CHECKLIST.md` to close the remaining demo tasks.
5. Execute `DEMO_RUNBOOK.md` once end-to-end.
6. Record the 10-minute video using `VIDEO_SCRIPT_10MIN.md`.
7. Use archived packaging docs only if you explicitly need one-off hiring collateral outside the main walkthrough flow.

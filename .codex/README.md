# ShelfOps Planning

This directory is the planning source of truth for ShelfOps.

Files:

- `ROADMAP.md` — the current-to-standout product + engineering specification
- `TASKS.json` — the phased implementation plan with acceptance criteria

Truth hierarchy:

1. `.codex/ROADMAP.md`
2. `.codex/TASKS.json`
3. current code and test-backed runtime behavior

Superseded:

- `.claude/`
- demo-specific planning docs used as prior roadmap surfaces

Use the codebase to determine what already exists.
Use `.codex` to determine what should be built next.
